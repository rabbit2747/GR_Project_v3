## 1. 개요

### 1.1 정의

**GR Atlas**는 IT 인프라를 **지도처럼 시각화**하는 다이나믹 인터랙티브 맵(Dynamic Interactive Map)입니다.

```
┌─────────────────────────────────────────────────────────────┐
│                        GR CORE                              │
├───────────────────┬───────────────────┬─────────────────────┤
│      GR DB        │   GR Framework    │      GR Atlas       │
│                   │                   │                     │
│  "무엇이 있는가"   │  "어떻게 분류할까" │   "어떻게 보여줄까"  │
│                   │                   │                     │
│  • IT 제품 정보    │  • 3D 좌표 체계   │  • 지도처럼 보기     │
│  • 취약점(CVE)    │  • 관계 규칙      │  • 공격 흐름 시각화  │
│  • 공격 기법      │  • 보안 정책 규칙  │  • 직관적 이해      │
└───────────────────┴───────────────────┴─────────────────────┘
```

### 1.2 위치

> **Atlas는 "제품"이 아니라 "기능(Feature)"**

- GR Core의 공통 시각화 컴포넌트
- 자동화 진단, IaC, Edu 등 여러 제품에 **내장**되는 엔진

### 1.3 지도 비유

```
일반 지도                          GR Atlas
─────────────                      ─────────
• 지형, 도로, 건물                 • Layer, Zone, 서버
• 교통 흐름                        • 데이터 흐름
• 위험 지역 표시                   • 취약점 위치 표시
• 최적 경로 안내                   • 공격 경로 시각화
```

## 2. 핵심 기능

### 2.1 인프라 맵 시각화

GR Framework의 3D 분류 기준(Layer, Zone, Sector)을 기반으로 인프라를 **기업 네트워크 구성도**처럼 시각화합니다.

```
┌─────────────────────────────────────────────────────────────┐
│                      GR Atlas 화면                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [인터넷]                                                  │
│      │                                                      │
│      ▼                                                      │
│   ┌──────┐     ┌──────┐     ┌──────┐                       │
│   │ WAF  │────▶│ API  │────▶│  DB  │                       │
│   │ ⚠️   │     │ 🔴   │     │ ✅   │                       │
│   └──────┘     └──────┘     └──────┘                       │
│   Zone 1       Zone 2       Zone 3                         │
│                                                             │
│   🔴 Critical(3)  ⚠️ Warning(5)  ✅ Safe                   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 인터랙티브 탐색 (Drill-Down)

시각화된 맵의 구성요소를 클릭하면 계층적으로 상세 정보를 탐색할 수 있습니다.

```
[인프라 구성요소 클릭]
        ↓
[설치 가능한 제품/소프트웨어 리스트]
        ↓
[제품 클릭 → MITRE Tactic 리스트]
        ↓
[Tactic 클릭 → Technique 리스트]
        ↓
[Technique 클릭 → 상세 기술/페이로드/방어법]
```

**MITRE ATT&CK과의 차별점**:
- MITRE: Tactic → Technique에서 끝
- GR Atlas: Technique → **세부 기술, 페이로드, 우회 기법**까지 drill-down

### 2.3 취약점/공격 흐름 시각화

CVE, 해킹 시나리오, 개별 기술을 검색하면 맵에 **하이라이트**로 흐름을 표시합니다.

```
[검색: CVE-2021-44228 (Log4Shell)]
        ↓
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   [인터넷] ══🔴══▶ [WAF] ══🔴══▶ [WAS] ══🔴══▶ [내부망]    │
│                      │            │                         │
│                      │     ┌──────┘                         │
│                      │     ▼                                │
│                      │   [LDAP] ══🔴══▶ [공격자 서버]       │
│                      │                                      │
│   🔴 = 공격 경로 하이라이트                                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 보안 상태 대시보드

기업의 보안 설정을 대입하면 현재 **취약한 구성요소**를 색상으로 표시합니다.

```
Zone/Layer별 보안 점수:

Zone 1 (Perimeter):  ████████░░  80%
Zone 2 (Application): ████░░░░░░  40%  ⚠️ 주의
Zone 3 (Data):        █████████░  90%
Zone 4 (Management):  ██████████  100%

취약 구성요소:
• API Server (Zone 2) - 3개 Critical CVE
• Legacy App (Zone 2) - 패치 미적용
```

## 3. 활용 시나리오

### 3.1 자동화 진단 (DAST)

```
입력: 고객사 인프라 정보 (스캐닝 결과)
        ↓
    GR Framework로 분류
        ↓
    취약점 자동 매칭 (GR DB)
        ↓
출력: 상세 진단 보고서 + Atlas 시각화
```

### 3.2 보안 컨설팅

- 기존: 텍스트 보고서 → 고객 이해 어려움
- GR Atlas: **시각적으로 문제점 표시** → 임원/개발자/관리자 모두 이해

```
"Zone2의 API 서버에서 SQLi 취약점이 발견되었습니다"
        ↓
[Atlas에서 해당 서버가 빨간색으로 깜빡임]
[클릭하면 공격 경로와 영향 범위 표시]
```

### 3.3 보안 교육 (Edu)

- MITRE ATT&CK과 매핑하여 **전체 해킹 흐름** 시각적 학습
- 학습자별 실력 측정 및 취약 분야 파악
- 커리큘럼 추천

### 3.4 Red Team Playbook

- 모의해킹 시나리오 시각화
- 침투 경로 계획 및 문서화
- 컨설턴트 역량 개발 (부족한 부분 체크)

## 4. 기술 사양

### 4.1 데이터 소스

```yaml
data_sources:
  gr_db:
    - 인프라 컴포넌트 정보
    - 취약점 (CVE)
    - 공격 기법 (MITRE 매핑)
    - 방어 기법
    
  gr_framework:
    - 3D 좌표 (Layer, Zone, Sector)
    - 관계 규칙
    - 보안 정책 규칙
    
  external:
    - 고객 인프라 스캔 결과
    - 보안 설정 정보
```

### 4.2 시각화 레이어

```yaml
visualization_layers:
  base_layer:
    - 인프라 토폴로지 (네트워크 구성도)
    - Zone 경계선
    - Layer 구분
    
  overlay_layers:
    - 취약점 히트맵
    - 공격 경로 애니메이션
    - 데이터 플로우
    - 보안 점수 그라데이션
    
  interactive:
    - 클릭 → 상세 정보 패널
    - 호버 → 툴팁
    - 검색 → 하이라이트
    - 필터 → 특정 Zone/Layer만 표시
```

### 4.3 연동 기술 (예정)

```yaml
backend:
  - Neo4j: 그래프 데이터 쿼리 (공격 경로 분석)
  - PostgreSQL: 원자 데이터 조회
  
frontend:
  - D3.js / Cytoscape.js: 그래프 시각화
  - Three.js: 3D 렌더링 (선택)
  - React/Vue: UI 프레임워크
  
api:
  - GraphQL: 유연한 쿼리
  - REST: 표준 CRUD
```
