### 2. Zone 0 (Management Zone) 정의 문서

**파일**: `02_차원2_Security_Zone/Zone_0_Management/00_Zone_0_정의.md`

#### 주요 내용:

```yaml
정의:
  - 시스템 전체의 통제, 모니터링, 보안 관리
  - 신뢰 수준: Highest
  - 공격 노출도: Lowest

배치 구성요소:
  Very Common:
    - SIEM (Splunk, QRadar, Sentinel)
    - IAM (Keycloak, Okta, Auth0)
    - Secrets Management (HashiCorp Vault)
    - Monitoring (Prometheus + Grafana)
  Common:
    - ITSM (ServiceNow)
    - Configuration Management (Consul)
    - PKI (Let's Encrypt, DigiCert)

보안 정책:
  - 네트워크: 인터넷 직접 노출 절대 금지
  - 인증: MFA 필수, 특권 계정 관리 (PAM)
  - 로깅: 모든 활동 기록, 1년 이상 보존
  - 데이터: 저장/전송 시 암호화 (AES-256, TLS 1.3)

실전 예시:
  - Startup: ~$500/월 (SaaS 중심)
  - Mid-size: ~$4,000/월 (하이브리드)
  - Enterprise: ~$75,000/월 (완전 자체 구축)

운영 프로세스:
  - 일일: 보안 리뷰, 시스템 헬스 체크
  - 주간: 권한 검토, 취약점 스캔
  - 월간: 패치 관리, DR Drill
```

**통계**:
- 총 라인 수: ~750줄
- 주요 섹션: 13개

### 3. Zone 1 (Perimeter Zone) 정의 문서

**파일**: `02_차원2_Security_Zone/Zone_1_Perimeter/00_Zone_1_정의.md`

#### 주요 내용:

```yaml
정의:
  - 외부 트래픽의 첫 번째 방어선
  - 신뢰 수준: Low
  - 공격 노출도: Highest

배치 구성요소:
  Very Common:
    - WAF (ModSecurity, AWS WAF, Cloudflare WAF)
    - Load Balancer (NGINX, HAProxy, AWS ALB/NLB)
    - CDN (Cloudflare, Akamai, CloudFront)
  Common:
    - Reverse Proxy (NGINX, Apache, Traefik)
    - DDoS Protection (Cloudflare, AWS Shield)

보안 정책:
  - 네트워크: HTTP/HTTPS만 허용, Zone 3 직접 접근 차단
  - 인증: API Key/Token 검증, IP 기반 통제, Bot 탐지
  - 로깅: 모든 요청/응답 로깅, 30일 보존
  - 데이터: TLS 1.2+ 필수, HSTS 활성화

WAF 룰셋:
  - SQL Injection, XSS, RCE, LFI
  - Command Injection, OWASP Top 10

실전 예시:
  - Startup: ~$50/월 (Cloudflare Free)
  - Mid-size: ~$600/월 (Cloudflare Pro + AWS WAF)
  - Enterprise: ~$17,000/월 (Akamai + Imperva + F5)

인시던트 대응:
  - DDoS 공격: 5-15분 대응
  - Zero-Day 공격: 1-4시간 대응
  - TLS 인증서 만료: 즉시 또는 1-2시간
```

**통계**:
- 총 라인 수: ~800줄
- 주요 섹션: 14개

### 4. Zone 2 (Application Zone) 정의 문서

**파일**: `02_차원2_Security_Zone/Zone_2_Application/00_Zone_2_정의.md`

#### 주요 내용:

```yaml
정의:
  - 비즈니스 로직 실행 및 API 제공
  - 신뢰 수준: Medium
  - 공격 노출도: Medium

배치 구성요소:
  Very Common:
    - Backend API (Node.js, Spring Boot, Django)
    - App Server (Tomcat, Gunicorn, Puma)
    - Message Queue (Kafka, RabbitMQ, AWS SQS)
  Common:
    - API Gateway (Kong, AWS API Gateway)
    - Service Mesh (Istio, Linkerd)
    - Cache (Redis, Memcached)

보안 정책:
  - 네트워크: Zone 1에서 검증된 트래픽만, Zone 4 직접 접근 금지
  - 인증: JWT Token 검증, OAuth 2.0, mTLS (Service-to-Service)
  - 로깅: API 호출, 비즈니스 로직 실행, 90일 보존
  - 데이터: PII 암호화, ORM 사용, 입력 검증

실전 예시:
  - Startup: ~$190/월 (모놀리식)
  - Mid-size: ~$2,000/월 (마이크로서비스)
  - Enterprise: ~$13,000/월 (100+ 서비스, Kubernetes)

데이터 흐름:
  - 동기: User → Zone 1 → Zone 2 → Zone 3
  - 비동기: Event-Driven (Kafka)

인시던트 대응:
  - 서비스 장애: 5-15분
  - Database 연결 고갈: 10-30분
  - 메모리 누수: 1-4시간
```

**통계**:
- 총 라인 수: ~800줄
- 주요 섹션: 14개

### 5. Zone 3 (Data Zone) 정의 문서

**파일**: `02_차원2_Security_Zone/Zone_3_Data/00_Zone_3_정의.md`

#### 주요 내용:

```yaml
정의:
  - 민감한 데이터 저장 및 관리
  - 신뢰 수준: High
  - 공격 노출도: Low

배치 구성요소:
  Very Common:
    - Relational Database (PostgreSQL, MySQL, Oracle)
    - NoSQL Database (MongoDB, Cassandra, DynamoDB)
    - Object Storage (AWS S3, MinIO)
  Common:
    - Cache (Redis, Memcached)
    - Data Warehouse (Snowflake, Redshift, BigQuery)
    - Backup System (Veeam, AWS Backup)

보안 정책:
  - 네트워크: Zone 2에서 인증된 연결만, Zone 1/4 직접 접근 절대 차단
  - 인증: 애플리케이션별 전용 계정, TLS 클라이언트 인증서
  - 로깅: 모든 쿼리 기록, 1년 이상 보존, 변조 방지 (WORM)
  - 데이터: 저장/전송 시 암호화 (AES-256-GCM, TLS 1.3), PII 컬럼 레벨 암호화

백업 정책:
  - 빈도: 일일 Full + 매시간 증분
  - 암호화: AES-256 (백업 전)
  - RTO: 4시간, RPO: 1시간

실전 예시:
  - Startup: ~$253/월 (RDS, S3)
  - Mid-size: ~$5,400/월 (자체 구축 HA)
  - Enterprise: ~$65,000/월 (Oracle RAC, Multi-DC)

인시던트 대응:
  - Database 장애: 5-15분 (자동 Failover)
  - 데이터 손상: 1-4시간
  - 랜섬웨어 공격: 4-24시간
```

**통계**:
- 총 라인 수: ~800줄
- 주요 섹션: 14개

### 6. Zone 4 (Endpoint Zone) 정의 문서

**파일**: `02_차원2_Security_Zone/Zone_4_Endpoint/00_Zone_4_정의.md`

#### 주요 내용:

```yaml
정의:
  - 최종 사용자 및 디바이스 영역
  - 신뢰 수준: Variable (사용자/디바이스별 상이)
  - 공격 노출도: High

배치 구성요소:
  Very Common:
    - 웹 브라우저 (Chrome, Safari, Firefox, Edge)
    - 모바일 앱 (iOS App, Android App)
  Common:
    - Desktop 앱 (Electron, .NET, Qt)
    - VPN Client (OpenVPN, WireGuard)
  Rare:
    - IoT 디바이스 (Sensors, Gateways)
    - Kiosk 단말

보안 정책:
  - 네트워크: HTTPS만 허용, Zone 2/3 직접 접근 절대 금지
  - 인증: MFA 필수, 디바이스 인증 (Certificate), 지속적 검증
  - 로깅: 로그인/로그아웃, 페이지 방문, 180일 보존
  - 데이터: 로컬 저장 최소화, AES-256 암호화, 자동 삭제

Zero Trust:
  - 디바이스 신뢰 평가 (OS 버전, Security Patch)
  - Conditional Access (위치, 시간, 네트워크)
  - Step-Up Authentication (민감한 작업 시 재인증)

실전 예시:
  - Startup: ~$170/월 (BYOD 허용)
  - Mid-size: ~$1,300/월 (MDM 도입)
  - Enterprise: ~$28,000/월 (회사 소유만, Zero Trust)

인시던트 대응:
  - 디바이스 분실: 즉시 (Remote Wipe)
  - 악성 앱 설치: 5-30분
  - 피싱 공격: 10-60분
```

**통계**:
- 총 라인 수: ~800줄
- 주요 섹션: 14개

## 🎯 달성 효과

### 1. Security Zone 프레임워크 완성

**변경 전**:
- Zone 개념만 언급 (개요 문서)
- 상세 정의 부족
- 실무 적용 가이드 없음

**변경 후**:
- 5-Zone Model 완전 문서화
- Zone별 배치 패턴, 보안 정책 상세화
- 3단계 실전 예시 (Startup → Mid-size → Enterprise)

### 2. 보안 정책 일관성 확보

**변경 전**:
- Zone 간 정책 불명확
- 경계 통제 규칙 부재
- 트래픽 흐름 정의 없음

**변경 후**:
- Zone별 네트워크/인증/로깅 정책 명시
- Zone 간 허용/차단 규칙 명확화
- 정상/공격 트래픽 흐름 시각화

### 3. 실무 적용 가능성 향상

**변경 전**:
- 이론적 개념만
- 실제 구현 예시 부족
- 비용 정보 없음

**변경 후**:
- 3단계 배치 예시 (Startup, Mid-size, Enterprise)
- 실제 제품명 및 가격 정보
- 인시던트 대응 시나리오

### 4. Zero Trust 원칙 적용

**변경 전**:
- Zero Trust 언급만
- 구체적 구현 방법 부족

**변경 후**:
- Zone별 Zero Trust 적용 방법
- 지속적 검증, Conditional Access
- 디바이스 신뢰 평가 프레임워크
